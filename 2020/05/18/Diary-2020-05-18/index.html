Posted on 2020-05-18 In Diary

あなたはソフトウェア開発に対する哲学を持っていらっしゃいますか?

  • 結論として言いたいテーマ : ソフトウェア開発の際、Google検索で素早く開発するのではなく、
    海外の有名シニアエンジニアの書籍を時々読んで、ソフトウェアの新しい地平を開くエンジニアとして名を残したい。 私は

for FullStack Developer

  • 私はフルスタック·デベロッパーを目指している。

  • Software Developmentって、今やその規模と分野が大きくなりすぎて、
    各パートを担当するエンジニアやプログラマーの役割が分かれている。

  • 実は私はこのような部分が少し気になるものがある。
    バックエンドの開発者なら、ソフトウェア開発だけでなく運営まで責任を尽くさなければならないと思う。
    (もちろん、最近にはDevOpsエンジニアという新しい職群が設けられているが)

  • 私は最近アメリカのAmazonで有名なIT書籍を読んでいる。
    1冊目は、Linuxに対して非常に高い評価を受けている本で、
    2冊目は、DevOpsエンジニアのためのPython本で、
    3冊目は、アーキテクトのためのPython本だ。

  • 私もその時々に開発するとき、Google検索で簡単にLinux環境での開発プロジェクトを何度かやったことがある。
    ただ、私はこれらの開発プロセスで心配になったりする。
    ソフトウェアの開発について、私は個人的に哲学が必要だと思う。

  • 私はおよそ200人のプログラマー、エンジニアに会った。
    その中では本当に世界的にすごいエンジニアもいたが、ほとんどは職業としてのプログラマーが多かった。
    もちろん私が彼らより高い水準の開発実力を持っていると自負できるわけではない。
    (ずいぶん生意気な態度だと思う。)

  • しかし、ソフトウェア的に新しい地平を開くだけのレベルのエンジニアは珍しかった。
    私はその差が自分のソフトウェア開発に対する哲学の有無だと考え、時にはそれが堅物な態度で周囲の人々を困らせることもあると思うが、それでも自分だけのソフトウェア開発に対する哲学がなければならないともう一度強調したい。

  • それで私はGoogle検索を後にして、現在は有名ソフトウェア書籍を何度も読んでいる。
    多分このようなきっかけを与えたのは私がMITで授業を受けたからだと思う。

  • 私は韓国でプログラミングを学び、韓国ではプログラミングが上手だと認められるためにはアルゴリズムテストで高い点数をもらわなければならなかった。 (もちろんこれは大学の友達同士での話だ。)

  • 当時、私は本当にCQに夢中になっていた。
    韓国内のC++アルゴリズム問題の80%以上は解いていたようだ。
    そして、C++アルゴリズム評価で上位2%内に入るという評価を受けていた。
    それでなのかすごい自信感を持ってMITの授業を受けたけどSICPという科目で
    一言で言えば、アルゴリズムテストのようなテクニックだけでは現代プログラミングに到達するまでの数学者あるいはコンピュータ科学者の哲学に到達できなかったということだった。

  • その時、本当に深く感じた。
    さらには記憶喪失になり、ほとんどC++について忘れるほどだった。
    もちろん現在はとても健康だ。 もちろんC++はそれ以来触っていないので全部忘れたと思う。
