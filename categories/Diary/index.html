Diary-2020-06-20

最近悟った考え

  • 私は強く言いたい。 すべての人はプログラミングを学ばなければならないと思う。
  • 結論：
    • プログラミングを学ぶことで、自分の性格を知ることができ、
    • どのような人々と働きたいか、
    • そしてそのようなグループの中で、どのようなシナジー的な成果が達成できるかも分かるようだ。

過去の私

  • 私はJava, Pythonが私のプログラミングランゲージだった.
  • 理由としては、Javaのシェアが高く、ローレベルランゲージであったことから、大規模なSI環境でも高い完成度と信頼性のランゲージであったからだと思う。
  • 特に、Spring framework はMVC モデリングの頂点であったと考える。

現在の私

  • 近年では、JavaScript、Pythonの目覚ましい発展と強力なライブラリにより、
    ローレベル言語に匹敵するレベルになっていると考えている。
  • JavaScript : React + Node + AWS(Azure, GCP)
  • Python : Django API
  • すごく面白くて面白い。久しぶりに感じるプログラミングに対する熱情。
  • スクリプトランゲージの変化が早すぎるため、過去とは本当に変わったといってもおかしくない。

未来の私

  • JavaScript、Pythonフルスタックエンジニアとして働きたい。

  • 既に新入レベルのJavaScriptフルスタック開発者として競争力があると思う。 残念な点があるとしたら、フロントエンドなデザイン感覚がまだ足りない。 コンポーネントとデザイン的な試行錯誤がちょっと必要だ。
    ｄｉｖとの戦いが再び始まるかも知れない。

  • PythonではDjango-Server APIの開発が可能である。
    Flaskにするアーキテクチャの勉強をもっとしなければならない。 完成度がまだ低い。

MacUserはUpdateが怖い(?)

  • 私は MacBookProを使っていて、Updateはかなり怖いと思われていた(?)w
  • 実は、どんなProgrammingLanguageでも、FrameWorkはあるので、簡単にPortfolioを作るのができるんだ。
  • でも、MacはUpdateによる設定が急に変わって、それをまた修正するのがかなり時間がかかるんだ。

でも、Codingは楽しい

  • Coronaで、世の中は止まったかもしれないんですが、自分はCodingの勉強に夢中になっていた。
  • 引っ越ししたばかりで、ネットが出来なくてPortfolioを作る事がかなり遅くなってしまったが
  • 自分に合う働き先はどこにでもあると思う。 頑張ろう！

今日もカフェで、ほぼ10時間コーディングしている。

  • 問題はアップデートによる修正でかなり頭を抱えている。

あなたはソフトウェア開発に対する哲学を持っていらっしゃいますか?

  • 結論として言いたいこと : 海外の有名なシニアエンジニアの書籍を読もう。

  • Goolge検索を通じた開発もいいが、シニアエンジニアたちの経験の中から得た悟りが書籍の中にあると思う。

  • 私はフルスタック·デベロッパーを目指している。

  • Software Developmentって、今やその規模と分野が大きくなりすぎて、
    各パートを担当するエンジニアやプログラマーの役割が分かれている。

  • 実は私はこのような部分が少し気になるものがある。
    バックエンドの開発者なら、ソフトウェア開発だけでなく運営まで責任を尽くさなければならないと思う。
